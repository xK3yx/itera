# Itera

Itera is an AI-powered personalized learning roadmap generator. Fill in your profile вЂ” current role, target role, tech stack, experience вЂ” and Itera generates a structured, phase-by-phase learning roadmap with curated free and paid resources for every topic, time estimates split by learning path, and a semantic knowledge base used to validate your progress logs.

## Features

### Roadmap Generation
- **Structured roadmaps** вЂ” 3вЂ“4 phases, each broken into skill areas and topics, generated by a local LLM (Ollama) with Groq as automatic fallback
- **Per-topic resources** вЂ” YouTube playlists/videos, freeCodeCamp, Coursera, and Udemy links attached to every topic
- **Resource cache** вЂ” fetched resources are stored in the database by search query and reused across roadmaps, eliminating redundant API calls
- **Free / Paid / Avg time estimates** вЂ” each topic and the overall roadmap shows hours broken down by free path, paid path, and average
- **Background knowledge base** вЂ” a rich semantic KB (what it is, subtopics, validation keywords) is generated asynchronously after the roadmap is returned, so you're never waiting on it

### Progress Tracking
- **Semantic log validation** вЂ” describe what you learned in plain text; the backend uses fuzzy matching against the topic's knowledge base to decide if it counts
- **Enrollment & progress bar** вЂ” enroll in a roadmap to start tracking; per-topic completion and an overall percentage shown in real time
- **LLM call logging** вЂ” every LLM call is logged to the database with latency, token counts, model used, and hallucination flags

### UX
- **Typewriter reveal animation** вЂ” when a roadmap loads, phases are revealed one by one with a letter-by-letter typewriter effect on each phase title
- **Role dropdowns** вЂ” current role and target role fields use a combobox with 27 preset roles and free-text input for anything not in the list
- **Animated generating screen** вЂ” 5-step progress animation with elapsed timer while the roadmap is being built (~1вЂ“2 minutes)
- **Dark / Light theme**
- **JWT authentication**

## Getting Started

### Prerequisites
- Docker Desktop
- Ollama running locally with a model pulled (e.g. `ollama pull llama3.1`) вЂ” **or** a [Groq API key](https://console.groq.com) as fallback (free tier available)

### Run with Docker

```bash
git clone https://github.com/xK3yx/itera.git
cd itera/itera-backend

# Copy the env file and configure your keys
cp .env.example .env

docker compose up --build -d
```

Open **http://localhost** in your browser.

Alembic migrations run automatically on container start вЂ” no manual step needed.

### Environment Variables

```env
# App
APP_NAME=Itera
SECRET_KEY=your-secret-key
DEBUG=false

# Database
POSTGRES_USER=itera_user
POSTGRES_PASSWORD=itera_password
POSTGRES_DB=itera_db
DATABASE_URL=postgresql+asyncpg://itera_user:itera_password@db:5432/itera_db

# Redis
REDIS_URL=redis://redis:6379/0

# LLM вЂ” Ollama (primary)
OLLAMA_BASE_URL=http://host.docker.internal:11434
OLLAMA_MODEL=llama3.1

# LLM вЂ” Groq (fallback when Ollama is unreachable)
GROQ_API_KEY=your-groq-api-key

# YouTube Data API v3 (optional вЂ” falls back to search URLs if not set or quota exceeded)
YOUTUBE_API_KEY=your-youtube-api-key

# JWT
JWT_SECRET_KEY=your-jwt-secret
JWT_ALGORITHM=HS256
JWT_EXPIRE_MINUTES=1440
```

When running with Docker, `DATABASE_URL` and `REDIS_URL` use internal service hostnames automatically.

## API Endpoints

### Auth & Users
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/auth/register` | Register new user |
| POST | `/api/v1/auth/login` | Login, returns JWT |
| GET | `/api/v1/auth/me` | Get current user |
| PUT | `/api/v1/users/profile` | Update profile |

### Roadmaps
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v3/roadmaps/generate` | Generate a new roadmap |
| GET | `/api/v3/roadmaps/` | List all roadmaps for current user |
| GET | `/api/v3/roadmaps/{id}` | Get roadmap by ID |
| DELETE | `/api/v3/roadmaps/{id}` | Delete roadmap |

### Progress
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v3/roadmaps/{id}/enroll` | Enroll in a roadmap |
| GET | `/api/v3/roadmaps/{id}/enrollment` | Get enrollment + completed topics |
| POST | `/api/v3/roadmaps/{id}/progress/{topic_id}` | Submit a progress log entry |

### Knowledge Base & Admin
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v3/roadmaps/{id}/kb` | Get knowledge base for a roadmap |
| GET | `/api/v3/admin/llm-logs` | View LLM call audit log |

```bash
# Check resource cache
docker exec itera_postgres psql -U itera_user -d itera_db -P pager=off \
  -c "SELECT search_query, hit_count, jsonb_array_length(resources) AS resources FROM resource_cache ORDER BY hit_count DESC;"

# Stop
docker compose down

# Full reset (wipes database volume)
docker compose down -v
```

## How Generation Works

1. **Structure** вЂ” one LLM call produces phases and skill areas
2. **Topics** вЂ” one LLM call per skill area (run in parallel)
3. **Resources** вЂ” for each topic, check `resource_cache` first; on miss, fetch from YouTube API (falls back to search URL on quota exceeded), freeCodeCamp, Coursera, and Udemy, then store in cache
4. **Hours** вЂ” free path hours stamped on each topic; paid = free Г— 0.7; average shown in UI
5. **Persist** вЂ” roadmap saved to database and returned to user (~1вЂ“2 minutes total)
6. **KB** вЂ” knowledge base generated in background (asyncio task) after response is returned; used later for semantic progress matching
